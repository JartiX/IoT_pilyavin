# Калибровка движения колёсной платформы через Bluetooth

## Обзор

Прошивка позволяет откалибровать движение колёсной платформы **без перепрошивки** микроконтроллера. Все настройки подбираются в реальном времени через Bluetooth-соединение с помощью геймпад-интерфейса мобильного приложения.

Калибровка решает три ключевые проблемы:
1. **Неизвестная привязка пинов** — какая комбинация `DIR1`/`DIR2` вызывает какое движение (вперёд, назад, поворот).
2. **Разброс скоростей моторов** — колёса могут вращаться с разной скоростью, из-за чего машинка уходит в сторону.
3. **Нелинейность поворотов** — из-за инерции и разгона время поворота на 360° не равно четырёхкратному времени поворота на 90°.

Результат калибровки сохраняется в EEPROM и автоматически загружается при следующем включении.

## Аппаратная часть

| Компонент           | Описание                                    |
|---------------------|---------------------------------------------|
| Arduino Uno         | Микроконтроллер                             |
| Motor Shield (L298N)| Драйвер моторов, пины 4 (DIR1), 5 (SPD1), 6 (SPD2), 7 (DIR2) |
| HC-05               | Bluetooth-модуль, подключён к пинам 2 (RX) и 3 (TX) через SoftwareSerial |
| 2× DC-мотора        | Приводы колёс                               |
| Батарейный блок     | Автономное питание платформы                |

### Схема подключения HC-05

```
HC-05 TXD  →  Arduino Pin 2 (SoftwareSerial RX)
HC-05 RXD  →  Arduino Pin 3 (SoftwareSerial TX)  [через делитель 1к/2к]
HC-05 VCC  →  5V
HC-05 GND  →  GND
```

> **Важно:** Пин RXD модуля HC-05 работает на уровне 3.3 В. Используйте резистивный делитель напряжения (1 кОм + 2 кОм) на линии TX Arduino → RXD HC-05.

### Схема подключения Motor Shield

```
DIR1  = Pin 4   (направление мотора A)
SPD1  = Pin 5   (ШИМ скорость мотора A)
SPD2  = Pin 6   (ШИМ скорость мотора B)
DIR2  = Pin 7   (направление мотора B)
```

## Логика работы

Прошивка реализует **конечный автомат с четырьмя фазами**. Переход между фазами — последовательный, по нажатию кнопки **Start (A)**.

```
┌────────────┐  Start   ┌──────────┐  Start   ┌─────────┐   Start  ┌──────────┐
│ DIRECTION  │ ──────→  │ BALANCE  │ ──────→  │  TURNS  │ ──────→  │  DRIVE   │
│            │          │          │          │         │          │          │
│ Привязка   │          │ Баланс   │          │ Время   │          │ Езда +   │
│ комбинаций │          │ скоростей│          │поворотов│          │сохранение│
└────────────┘          └──────────┘          └─────────┘          └──────────┘
                                                                      │ Pause
                                                                      ↓
                                                                (перекалибровка)
```

## Кнопки геймпада

Приложение отправляет один ASCII-символ при нажатии кнопки и символ `0` при отпускании.

| Кнопка      | Символ | Расположение              |
|-------------|--------|---------------------------|
| ↑           | `F`    | D-pad (левая сторона)     |
| ↓           | `B`    | D-pad                     |
| ←           | `L`    | D-pad                     |
| →           | `R`    | D-pad                     |
| ✕ (крестик) | `X`    | Правая сторона            |
| ○ (круг)    | `C`    | Правая сторона            |
| △ (треуг.)  | `T`    | Правая сторона            |
| □ (квадрат) | `S`    | Правая сторона            |
| Start       | `A`    | Нижняя панель             |
| Pause       | `P`    | Нижняя панель             |
| Отпускание  | `0`    | —                         |

## Фазы калибровки

### Фаза 1: DIRECTION — привязка направлений

**Цель:** определить, какая из четырёх комбинаций `(DIR1, DIR2)` соответствует каждому виду движения.

Существуют ровно 4 комбинации:

| Индекс | DIR1 | DIR2 |
|--------|------|------|
| 0      | LOW  | LOW  |
| 1      | HIGH | LOW  |
| 2      | LOW  | HIGH |
| 3      | HIGH | HIGH |

**Кнопки в этой фазе:**

| Кнопка      | Действие                                     |
|-------------|----------------------------------------------|
| D-pad ↑↓←→  | Выбрать движение для калибровки              |
| △ (T)       | Переключить на следующую комбинацию DIR1/DIR2|
| ✕ (X)       | Подтвердить текущую комбинацию               |
| □ (S)       | Остановить моторы                            |
| Start (A)   | Перейти к фазе BALANCE (когда все 4 настроены)|

**Алгоритм:**

1. Нажмите `↑` (вперёд). Система запустит моторы с первой свободной комбинацией.
2. Наблюдайте за платформой:
   - Машинка поехала **вперёд** → нажмите **✕** для подтверждения.
   - Машинка поехала **не туда** → нажмите **△** для переключения на следующую комбинацию.
3. Повторите для `↓` (назад), `←` (влево), `→` (вправо).
4. Когда все 4 движения привязаны — нажмите **Start**.

### Фаза 2: BALANCE — балансировка скоростей

**Цель:** добиться прямолинейного движения, компенсируя разницу моторов.

Параметр `balance` смещает ШИМ-значения: `SPD1 = 180 + balance`, `SPD2 = 180 - balance`.

| Кнопка      | Действие                                     |
|-------------|----------------------------------------------|
| ↑ (F)       | Запустить движение вперёд                    |
| ← (L)       | Усилить левый мотор (balance −5)             |
| → (R)       | Усилить правый мотор (balance +5)            |
| ✕ (X)       | Сбросить баланс в 0                          |
| □ (S)       | Остановить моторы                            |
| Start (A)   | Перейти к фазе TURNS                         |

**Процедура:**
1. Нажмите `↑` — машинка поедет вперёд.
2. Уводит влево → нажимайте `←`. Уводит вправо → нажимайте `→`.
3. Повторяйте до прямолинейного хода.
4. Нажмите **Start**.

### Фаза 3: TURNS — калибровка поворотов

**Цель:** подобрать длительность поворотов на 90°, 180°, 270° и 360°.

| Кнопка      | Действие                                     |
|-------------|----------------------------------------------|
| ↑/↓         | Переключить угол (90→180→270→360)            |
| ← (L)       | Уменьшить время на 25 мс                    |
| → (R)       | Увеличить время на 25 мс                    |
| ✕ (X)       | Тестовый поворот **вправо**                  |
| △ (T)       | Тестовый поворот **влево**                   |
| □ (S)       | Остановить моторы                            |
| Start (A)   | Перейти к фазе DRIVE                         |

**Процедура:**
1. Выберите угол кнопками `↑`/`↓`.
2. Нажмите `✕` для теста поворота вправо.
3. Недокрутил → `→`, перекрутил → `←`.
4. Повторите. Настройте все 4 угла.
5. Нажмите **Start**.

> Время 360° ≠ 4 × время 90° из-за инерции и разгона.

### Фаза 4: DRIVE — свободное управление

Все настройки применены. Платформа управляется D-pad'ом.

| Кнопка      | Действие                                     |
|-------------|----------------------------------------------|
| ↑↓←→        | Езда                                         |
| □ (S) / `0` | Остановить моторы                            |
| ○ (C)       | **Сохранить** калибровку в EEPROM            |
| Pause (P)   | **Сброс** — начать калибровку заново         |

> При следующем включении Arduino калибровка загрузится из EEPROM автоматически, и система сразу перейдёт в режим DRIVE.

## Хранение данных в EEPROM

Калибровочные данные занимают 14 байт:

| Адрес  | Размер | Данные                                  |
|--------|--------|-----------------------------------------|
| 0      | 1      | Маркер валидности (`0xCA`)              |
| 1–4    | 4      | Индексы комбинаций для FWD/BWD/LFT/RGT |
| 5      | 1      | Баланс скоростей (смещённый на +50)     |
| 6–13   | 8      | Время поворотов: 4 × `uint16_t`         |

При загрузке проверяется маркер `0xCA`. Если данные повреждены или отсутствуют, калибровка начинается с нуля.

## Сборка и загрузка

1. Откройте `calibration.ino` в Arduino IDE.
2. Выберите плату **Arduino Uno** и порт.
3. Загрузите скетч.
4. Отключите USB — подключите автономное питание.
5. В мобильном приложении подключитесь к модулю HC-05 и выберите режим **Gamepad**.
6. Начинайте калибровку по инструкции.
